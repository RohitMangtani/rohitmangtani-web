- **Subtitle:** Why the future of project management is a terminal, a map, and a swarm of agents that ping you when they are lost

## Claim
Project management is not going to be managed by people managing people. It is going to be one person, a terminal, and a swarm of AI agents running in parallel. The bottleneck is not intelligence or speed. It is context. Whoever solves context management for agent swarms owns the next era of building.

## Section Structure

### 1. The Shift Already Happened
What PM looks like right now for someone running agent swarms. Four agents at once, each sourcing a different aspect of a project. 100x the output. Not theoretical. Already the workflow. The job is not doing, it is directing.

### 2. The Bottleneck Is Context
The biggest problem with agent swarms. Each agent has a window. When it fills up, it forgets. Context compaction, memory drift, goal degradation. The human role becomes context maintenance.

References: Carlini's 16 parallel Claudes, Zach Wills' 8 rules, context compaction patterns.

### 3. The Three Layers
Future PM workspace:
- **The Black Box** (Terminal): Intent in, output out.
- **The Map** (Vibe Coding Map): Visual representation from creator's perspective. Reference Crawler.
- **The Manager** (Find My iPhone for Agents): Real-time dashboard, green/red status, ping on need.
